#### 难点

1. **可编辑树形表格**

做技能标准模块的时候有一个需求，是配置工种等级考核的知识点，需要用到树形表格加上行编辑功能，当时看了 `procomponents` 中的 `EditableProTable` 的 demo 挺符合要求就选择使用它，后来在用的时候发现它内置的方法表现效果跟预期不一致，在新增一行数据保存后不能将接口返回的数据 id 替换掉表格中对应的数据，然后在其仓库的 issue 上有看到同样的问题但是没有解决，后来就不用其内置的方法，改为调用它提供的 API 将新增、删除、保存数据以及将数据插入到哪个位置这些每一步都手动去操作

2. **大文件上传**（课件 ppt、pdf）

   1. 首先生成一个 `uuid`，用于标识文件，然后设置分片大小，计算文件需要切割成几片
   2. 然后利用**file 对象的 slice 方法**分割文件，另外携带 `chunkIndex` 参数，标识当前是第几个分片，另外还要带上同一个 `uuid` 用于标识是同一个文件，然后调用上传分片接口（并发上传）。
   3. 当所有分片上传完后**调用合成分片接口告诉后端合成分片**（根据 `uid`），接口调用成功后返回文件资源地址。
   4. 注意当分片上传或者分片合成接口调用失败时需要调用一个**失败回调接口**通知后端将刚刚上传的分片删除。

   优化点：

   - 利用 `spark-md5`，根据文件内容计算出 `hash` 值，下次上传同样的文件服务器可以根据 `hash` 判断是否存在相同资源。

3. **虚拟列表**(讲师列表)

   1. 首先在列表头部和尾部添加用于检测列表滚动的元素，然后创建`IntersectionObserver`实例对这两个元素进行监视
   2. 在 `callback` 中分别处理监视到头部元素和尾部元素出现在视窗的时候根据索引去取数据（最开始接口返回所有数据存起来）
   3. 往上拉的时候需要手动处理滚动条位置

## 简历上可能的问题

### 封装了什么自定义 Hook

1. **useTable**

基于 `ahook` 的 [useAntdTable](https://ahooks.js.org/zh-CN/hooks/use-antd-table) 二次封装，具备以下功能：

- 自动请求数据
- 内置查询、条件重置、分页、排序（需要与后端统一排序参数）和删除逻辑处理

2. **useList**

用于处理列表数据。

- 自动请求数据
- 内置查询、条件重置、分页等逻辑处理

### 组件封装

1. **Table 组件**

- 分页器样式
- 单选（数据选择回填）、多选（批量删除）逻辑

基于 `Antd` 的 `Table` 组件的，主要是将表格选择(`rowSelection`)和数据删除的一些逻辑，还有表格和分页器的一些定制样式封装成一个业务组件，然后配合自定义的`hook`：`useTable`（基于`ahook`的`useAntdTable`）使用

2. **SearchForm 组件**

- 布局样式
- 输入框、下拉框、时间选择等组件内置，通过传入数据控制渲染

基于 `Antd` 的 `Form` 表单组件进一步封装，传入一个对象数组，数组里的每一个对象都是一个表单项，其中主要的属性就是 `type`，它决定渲染一个怎样的表单项，比如 `input` 就是输入框，`select` 就是下拉选择框，然后其他的属性像 `label`，`name`，`rules` 就直接透传给 `Form.Item`，其他像 `antd` 上输入框和选择框的属性就直接透传到对应的组件上，然后因为一般是搭配表格使用的，所以像其他的参数像表单的提交重置都交给 `ahook` 中的 `useAntdTable` 处理

3. **SearchForm + Table + useTable 页面组件**

- 加入 `useTable` 处理数据请求逻辑

- 自定义 `hook`

一个定制页面组件，由于学习平台的后台管理系统的大部分页面都是上面表单下面表格的内容，所以将前面提到的表单组件和表格组件组合起来，配合自定义 hook 使用，能给每个页面文件减少很多重复代码

4. **表格弹窗**

将之前用 `SearchForm + Table + useTable` 封装的页面组件放到 `Modal` 里，处理好表格选择逻辑和表格与弹窗之间的组件通信

5. **表格项操作**

表格最后一列是操作列，一般有查看、编辑、删除等操作按钮，封装这个组件首先是样式的定制，超过三个按钮的话就要用省略号加浮窗（`Tooltip`）展示剩余的操作按钮，然后用封装的另一个权限按钮组件去渲染按钮

6. **权限按钮**

由于当时这个功能是在项目后期做的，当时系统上已经有很多按钮了，考虑到最小限度的修改代码，就基于 `Antd` 的 `Button` 进行二次封装，将权限判断的逻辑封装在组件内，然后将组件注册到 `Antd` 的 `Button` 上，原本按钮的属性直接透传，不需要做额外的变动，只需要多传一个 `code` 编码参数就可以判断权限
